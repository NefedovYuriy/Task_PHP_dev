1. Дыра в безопасности данных пользователей.   
Т.к. любой пользователь на стороне клиента может зайти в папку upload и заполучить доступ к ранее вложенным файлам.  
    1.1 После загрузки, CSV файлы не шифруются. 

    Решение:  
    *Нужно дать доступ к директории с файлами конкретному пользователю. 
    *Либо шифровать файлы при загрузке на сервер.  


2. Дыра в безопасности сервера.  
Нет ограничений / проверки файлов.  
    2.1 Файл может содержать в себе вирус / вредоносный скрипт.  
    2.2 Файл может быть слишком большого размера. 
    2.3 Нет ограничений на количество файлов. 

    Решение:  
    *Нужно валидировать файлы на размеры и содержимое.  
    *Также, нужно проводить валидацию на количество файлов.   


3. Дыра в оптимизации.  
Папка upload никак не отсортирована.  
    3.1 После небольшого промежутка времени найти в ней что-либо будет затруднительно.  
    3.2 Не отображается время загрузки файла. 
    3.3 Файлы не завязаны на пользователей, который их непосредственно вкладывает. 

    Решение: 
    *Нужно для каждого пользователя создавать отдельную папку с ID, где будут содержаться конекртно его файлы.
    *Нужно корректировать название файла в соответствии с ID пользователя и временем загрузки.
